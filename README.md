Introduction
中風患者與身心障礙者在日常生活中常面臨 動作控制不足 與 復健動機低落 的挑戰。傳統復健方式多以重複性動作為主，缺乏即時回饋與趣味性，導致患者參與度有限。
受到 Xbox 360 Kinect 互動遊戲 的啟發，本研究提出一個結合 遊戲引擎 與 深度攝影機骨架追蹤 的復健互動系統。研究目的如下：
- 建立一個 遊戲化復健平台，透過全身性互動提升患者參與度。
- 驗證系統能否在 患者同時佩戴輔具（如護具、支架） 的情況下，仍正確判斷人體運動狀態。
- 探討 不同深度攝影機型號 在骨架追蹤精度與穩定性上的差異，並提出最佳化選擇。

Methods
系統架構
- 遊戲引擎：選用 Godot 4.x，具備開源、跨平台與外部模組整合能力。
- 骨架追蹤：採用 Python 開源程式庫（參考 GitHub 開發者專案），進行關節點定位與動作分析。
- 資料串接：透過 Socket API 將骨架座標傳輸至遊戲引擎，映射為遊戲角色動作。
遊戲模組設計
- 快速找找看：訓練注意力與反應抑制。
- 記憶小舖：訓練工作記憶與順序記憶。
- 智慧路線：訓練規劃與問題解決能力。
- 尋寶探險：訓練視空間注意力與忽略側補償。
- 語意小廚房：訓練語言理解與語意分類。

Results
- 已完成：
- 系統架構設計與遊戲模組規劃
- Python 骨架追蹤模組測試（參考 GitHub 開源程式碼）
- 深度攝影機規格比較與採購決策

Discussion
- 全身性互動
- 預期 D456 能提供穩定的全身骨架追蹤，並支援多模組遊戲互動。
- 未來將進行 臨床測試，驗證其在不同動作與姿勢下的準確性。
- 輔具整合
- 患者常需佩戴護具或支架，這可能影響骨架偵測。
- 本研究將設計實驗，測試系統在不同輔具條件下的辨識率。
- 技術挑戰
- 近距離誤判：需透過濾波與演算法修正。
- 遠距離精度：需調整解析度與光源條件。
- 未來應用
- 與復健醫療單位合作，進行小規模臨床試驗。
- 發表於 Assistive Technology、ACM Transactions on Accessible Computing 等國際期刊。
- 延伸應用至 教育訓練、智慧家庭互動 等領域。

Conclusion
本研究提出一個結合 遊戲化設計 與 深度攝影機骨架追蹤 的復健互動系統。雖然目前專案仍停留在 D456 採購階段，但已完成系統架構設計與技術可行性分析。未來將持續推進 全身性互動測試 與 輔具整合實驗，期望能為中風與身心障礙患者提供更有效的復健工具。
